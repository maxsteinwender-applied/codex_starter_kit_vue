# Product Docs

## Purpose
This folder stores product-source artifacts for scope, prioritization, and feature definition.

## What Is Here
- `PRD.md` for vision, goals, and roadmap.
- `features/` for the feature index and individual feature specs.

## When To Use
Use these docs for product decisions, feature scoping, and handoffs to UX, frontend, and QA.

## Scope Boundary
Do not place technical stack constraints or global security policy files in this folder.

## Maintenance
Document changes along the standard workflow and keep status and acceptance criteria current.
